# Internal Linking System

## Philosophy

Curated, contextual, finite. Links are stored as explicit related-slug arrays in content data — never generated by keyword matching, never auto-linked on occurrence, never dumped as keyword lists in footers. Every link should be one a thoughtful editor would place.

## The Link Architecture by Page Type

**Service pages** link to: related services (2–4), relevant specialties, relevant solutions, relevant guides, the pricing resource (sidebar), and conversion CTAs. Rendered via RelatedLinks cards + sidebar modules.

**Specialty pages** link to: services used by that specialty (3–4), related specialties (2–3), relevant Texas locations (2–3), guides, and the assessment CTA. Educational disclaimer links to editorial pages.

**Location pages** link to: specialties relevant to that market (2–4), nearby locations (2–4), the services hub, and conversion CTAs. The locations hub links every city; cities cross-link within metro clusters via nearbyLocations.

**Solution pages** link to: the services that address the problem (2–3), related solutions (1–3), diagnostic guides.

**Guides** link to: relevant services, related solutions, and sibling guides via frontmatter arrays; body prose links contextually where genuinely helpful.

**Payer/software/cost/comparison pages** follow the same pattern with their type-appropriate targets.

**Hubs** list their children (ItemList schema) and are linked from header mega-menus (curated subset only), footer columns (curated subset), and the HTML sitemap (complete).

## Global Navigation Rules

- Mega menus show ≤15 curated links per hub + "view all" — never full inventories
- Footer shows 6–7 per category + hub links
- The HTML sitemap (/sitemap/) is the one complete-listing page, by design

## Anchor Text Rules

Descriptive and natural ("Denial management services", "Medical billing in Houston") — no exact-match stuffing, no "click here", no raw URLs in prose.

## Integrity Enforcement

Related-slug references across data modules are build-checked (typos fail the build). All rendered hrefs are checked against built pages post-build (broken links fail the QA script). When adding a page, add inbound links from 1–3 existing relevant pages — orphan pages are an editorial failure the checklist catches.

## Anti-Patterns (do not add)

- Auto-linking every occurrence of a keyword
- Giant related-links blocks (cap: 6 cards per section)
- Footer link farms beyond the curated columns
- Cross-linking unrelated content for "equity" — relevance only
